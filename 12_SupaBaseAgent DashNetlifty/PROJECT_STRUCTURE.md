# Структура проекта

Документация всех файлов в проекте Airtable Versioning Application.

## Корневые файлы

### `index.html`
Основной HTML файл приложения. Содержит базовую структуру страницы, подключение к React приложению через `src/main.tsx` и настройки viewport для адаптивности.

### `package.json`
Конфигурация npm проекта. Определяет зависимости, скрипты сборки и разработки, метаданные проекта. Содержит React, TypeScript, Vite и другие необходимые пакеты.

### `vite.config.ts`
Конфигурация сборщика Vite. Настраивает React плагин и оптимизацию зависимостей (исключает lucide-react из предварительной сборки).

### `tsconfig.json`
Основной файл конфигурации TypeScript. Ссылается на `tsconfig.app.json` и `tsconfig.node.json` для разделения настроек клиента и сервера.

### `tsconfig.app.json`
Конфигурация TypeScript для клиентского кода. Настраивает ES2020, JSX, строгую типизацию и модульную систему для браузерного окружения.

### `tsconfig.node.json`
Конфигурация TypeScript для Node.js окружения (Vite конфиг). Настраивает ES2022 и модульную систему для серверных инструментов.

### `tailwind.config.js`
Конфигурация Tailwind CSS. Определяет пути к файлам для сканирования классов и базовые настройки темы.

### `postcss.config.js`
Конфигурация PostCSS. Подключает Tailwind CSS и Autoprefixer для обработки стилей.

### `eslint.config.js`
Конфигурация ESLint для проверки качества кода. Настраивает правила для TypeScript, React hooks и React refresh.

### `netlify.toml`
Конфигурация для деплоя на Netlify. Определяет команды сборки, папку публикации и настройки функций (в данный момент не используется).

### `README.md`
Подробная документация проекта на русском языке. Содержит описание функций, технического стека, инструкции по настройке и использованию.

## Папка `src/`

### `src/main.tsx`
Точка входа React приложения. Создает корневой элемент, подключает стили и рендерит основной компонент App в StrictMode.

### `src/App.tsx`
Главный компонент приложения. Содержит основную логику UI, использует хук useRecords для управления состоянием и рендерит все основные компоненты интерфейса.

### `src/index.css`
Основной файл стилей. Подключает директивы Tailwind CSS (@tailwind base, components, utilities).

### `src/vite-env.d.ts`
Типы для Vite окружения. Подключает типы для работы с Vite в TypeScript.

## Папка `src/types/`

### `src/types/index.ts`
Определения TypeScript интерфейсов. Содержит типы для записей Airtable, API ответов, редактируемых полей и полей нового агента.

## Папка `src/services/`

### `src/services/api.ts`
Сервис для работы с Airtable API. Содержит класс ApiService с методами для получения, создания и удаления записей, управления версиями и обработки ошибок.

## Папка `src/hooks/`

### `src/hooks/useRecords.ts`
Кастомный React хук для управления состоянием записей. Содержит логику загрузки данных, редактирования, создания новых агентов, отмены изменений и обработки ошибок.

## Папка `src/components/`

### `src/components/RecordTable.tsx`
Компонент таблицы записей. Отображает список всех агентов в табличном формате с возможностью выбора, показывает состояния загрузки и пустые состояния.

### `src/components/RecordDetails.tsx`
Компонент детального просмотра и редактирования записи. Содержит формы для просмотра, редактирования существующих записей и создания новых агентов.

### `src/components/LoadingSpinner.tsx`
Компонент индикатора загрузки. Анимированный спиннер с настраиваемыми размерами (small, medium, large) для показа состояний загрузки.

### `src/components/ErrorNotification.tsx`
Компонент уведомлений об ошибках. Всплывающее уведомление с возможностью повторной попытки и закрытия, появляется в правом верхнем углу.

## Тестовые файлы

### `src/TestApi.tsx`
Компонент для тестирования подключения к Airtable API. Используется для отладки и проверки работоспособности API соединения.

### `src/api-small.ts`
Минимальный модуль для тестирования API. Содержит упрощенную функцию тестирования подключения к Airtable с логированием в консоль.

## Скрытые файлы

### `.env`
Файл переменных окружения (не отображается в проекте). Содержит секретные ключи API и конфигурацию, не должен попадать в Git.

### `.gitignore`
Список файлов и папок, игнорируемых Git (не отображается в проекте). Исключает node_modules, dist, .env и другие служебные файлы.

### `package-lock.json`
Файл блокировки зависимостей npm (не отображается в проекте). Фиксирует точные версии всех установленных пакетов для воспроизводимых сборок.

## Архитектура проекта

Проект следует модульной архитектуре:
- **Компоненты** - переиспользуемые UI элементы
- **Хуки** - логика состояния и побочных эффектов  
- **Сервисы** - взаимодействие с внешними API
- **Типы** - TypeScript определения для типобезопасности
- **Конфигурация** - настройки инструментов сборки и линтинга
