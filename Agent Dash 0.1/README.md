# Менеджер AI Агентов

Красивое, готовое к продакшену full-stack приложение для управления записями агентов с возможностями версионирования, использующее Airtable в качестве базы данных и Netlify Functions для безопасного доступа к API.

## 🚀 Возможности

- **Интеграция с Airtable**: Прямое подключение к Airtable API для управления данными
- **Система версионирования**: Автоматическое увеличение версии с группировкой по одинаковому p2_id
- **Интерактивный UI**: Чистый табличный интерфейс с выбором строк и детальным редактированием
- **Обновления в реальном времени**: Оптимистичные обновления UI с обработкой ошибок
- **Функция отмены**: Возможность отменить последние изменения
- **Адаптивный дизайн**: Оптимизирован для десктопа, планшета и мобильных устройств
- **Готов к продакшену**: Построен с TypeScript, современными React hooks и комплексной обработкой ошибок

## 🛠️ Технический стек

- **Frontend**: React 18 + TypeScript + Vite
- **Стилизация**: Tailwind CSS с кастомной дизайн-системой
- **База данных**: Airtable через REST API
- **Иконки**: Lucide React
- **Работа с датами**: date-fns

## 📋 Требования

- Node.js 18+ 
- Среда разработки Bolt
- Аккаунт Airtable с доступом к API

## 🔧 Настройка окружения

### Конфигурация Airtable

Приложение подключается к Airtable с предустановленными настройками. Для изменения подключения отредактируйте файлы:

| p2_variable_4 | string | Резервное поле        |

## 🔌 API Endpoints

Приложение использует прямые вызовы к Airtable API:

### GET к Airtable API
- Получает все записи, отсортированные по времени создания (новые первыми)
- Возвращает: `AirtableRecord[]`

### GET к Airtable API (конкретная запись)
- Получает детали конкретной записи
- Возвращает: `AirtableRecord`

### POST к Airtable API
- Создает новую запись с автоинкрементом версии
- Body: `{ fields: EditableFields }`
- Возвращает: `AirtableRecord`

### DELETE к Airtable API
- Удаляет конкретную запись (используется для функции отмены)
- Возвращает: `{ success: boolean }`

## 📖 Инструкции по использованию

### Описание интерфейса

#### Основные элементы

**Заголовок приложения**
- Иконка базы данных и название "Agent Manager"
- Подзаголовок "Manage your AI agents with version control"
- Кнопка "Создать агента" в правом верхнем углу для добавления новых агентов

**Таблица агентов (верхняя часть)**
- Заголовок секции "Все агенты"
- Таблица со следующими столбцами:
  - **Название агента**: Основное имя агента (p2_wf_name)
  - **Версия**: Номер версии в формате X.Y.Z (p2_wf_version)
  - **Функции**: Краткое описание функций агента (p2_variable_1)
  - **Вход**: Входные данные агента (p2_variable_2)
  - **Выход**: Выходные данные агента (p2_variable_3)
  - **Обновлено**: Время последнего обновления в относительном формате
- Выбранная строка выделяется синим цветом с левой границей
- Клик по любой строке выбирает агента для просмотра деталей

**Панель деталей записи (нижняя часть)**
- Заголовок "Детали записи" с кнопками управления справа
- Поля для просмотра и редактирования:
  - **Название агента**: Текстовое поле (обязательное при создании)
  - **Версия**: Только для чтения, автоматически генерируется
  - **Функции**: Многострочное текстовое поле для описания функций
  - **Вход**: Многострочное поле для входных данных
  - **Выход**: Многострочное поле для выходных данных
  - **Резервное поле**: Дополнительное поле для заметок

#### Кнопки управления

**В режиме просмотра:**
- **Edit**: Переключает в режим редактирования
- **Отменить изменения**: Появляется после сохранения, удаляет последнюю версию

**В режиме редактирования:**
- **Подтвердить**: Сохраняет изменения и создает новую версию

**В режиме создания нового агента:**
- **Добавить**: Создает нового агента с версией 1.0.0
- **Отмена**: Отменяет создание и возвращает к просмотру

#### Состояния интерфейса

**Загрузка**
- Анимированные скелетоны в таблице при первой загрузке
- Спиннеры на кнопках во время выполнения операций
- Блокировка кнопок во время обработки запросов

**Ошибки**
- Всплывающие уведомления в правом верхнем углу
- Кнопки "Retry" для повторной попытки
- Кнопка "Dismiss" для закрытия уведомления

**Пустые состояния**
- "No records found" при отсутствии данных
- "Select a record to view details" при отсутствии выбранной записи

### Просмотр записей
1. Записи отображаются в сортируемой таблице с новыми записями первыми
2. Кликните на любую строку для просмотра детальной информации
3. Выбранная запись выделяется синим акцентом

### Редактирование записей  
1. Нажмите кнопку **"Edit"** в секции деталей записи
2. Измените любые поля в интерфейсе формы
3. Нажмите **"Confirm"** для сохранения изменений
4. Автоматически создается новая версия с тем же `p2_id`

### Управление версиями
- Каждое сохранение создает новую запись с увеличенной версией
- Предыдущие версии остаются неизменными в базе данных
- Номера версий автоинкрементируются (например, 1.0.1 → 1.0.2)

### Отмена изменений
1. После подтверждения изменений появляется кнопка **"Undo Changes"**
2. Нажмите для удаления последней созданной записи
3. Предыдущая версия снова становится активной

### Создание нового агента
1. Нажмите кнопку **"Создать агента"** в правом верхнем углу
2. Заполните все необходимые поля в форме
3. Нажмите **"Добавить"** для создания нового агента
4. Новый агент создается с версией 1.0.0

### Обработка ошибок
- Сетевые ошибки отображают уведомление с опцией повтора
- Все сбои API показывают понятные пользователю сообщения об ошибках
- Валидация формы предотвращает пустые обязательные поля
